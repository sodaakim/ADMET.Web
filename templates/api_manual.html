API Introduction

This section provides an overview of our API, including its purpose and capabilities. Our API is designed to offer programmatic access to our platform, allowing developers to integrate our data and services into their applications.

Getting Started

Before you begin, you'll need to sign up for an API key. This key will authenticate your requests. Visit our developer portal to sign up and receive your API key.

Authentication

All API requests must be accompanied by your API key. Include your key in the request header as 'Authorization: Bearer YOUR_API_KEY'.

Endpoints

Our API provides several endpoints to access different types of data and perform various operations. Here is a list of available endpoints and their functions:

  • /analyze - Analyze SMILES strings and return ADMET characteristics.
  • /screening - Screen multiple SMILES strings in a single request.
  • /results - Retrieve results of previous analyses.

Rate Limits

To ensure fair usage, our API enforces rate limits. Standard users are limited to 100 requests per hour. Please contact us if you need higher limits for your application.

Examples

Here are some example requests to help you get started:

  • Analyze Request: curl -H "Authorization: Bearer YOUR_API_KEY" -X POST -d "smiles=Cc1ccccc1" https://api.yourservice.com/analyze
  • Screening Request: curl -H "Authorization: Bearer YOUR_API_KEY" -X POST -d "smiles=Cc1ccccc1\nCCO" https://api.yourservice.com/screening

Error Codes

Understanding error codes can help you troubleshoot issues. Here are some common codes:

  • 400 Bad Request - The request was unacceptable, often due to missing a required parameter.
  • 401 Unauthorized - No valid API key provided.
  • 429 Too Many Requests - Request rejected due to rate limit being exceeded.
